\section{Base de Datos y Modelos}

\subsection{Esquema Relacional}
El sistema utiliza un esquema relacional gestionado mediante SQLAlchemy y control de versiones de esquema con Alembic.

\subsubsection{Modelo de Usuarios (\texttt{User})}
\begin{itemize}
    \item \textbf{User}: Modelo centralizado con email como identificador principal. Almacena:
    \begin{itemize}
        \item \texttt{is\_active}, \texttt{is\_superuser}: Permisos base.
        \item \texttt{role}: Rol funcional (reader, moderator, admin).
        \item \texttt{is\_nsfw\_allowed}: Booleano para control parental/contenido sensible.
    \end{itemize}
\end{itemize}

\subsubsection{Modelo de Mangas (\texttt{Manga})}
La entidad central \texttt{Manga} se relaciona con:
\begin{itemize}
    \item \textbf{Estado:} FK a la tabla \texttt{Estado} (En emisión, Finalizado).
    \item \textbf{Demografía:} FK a \texttt{Demografia} (Seinen, Shonen).
    \item \textbf{Géneros/Tags:} M2M mediante tabla intermedia para categorización.
    \item \textbf{Cover:} Relación para almacenar la portada principal y portadas secundarias.
\end{itemize}

\subsubsection{Modelo de Estructura}
Tablas auxiliares para estandarización:
\begin{itemize}
    \item \texttt{Demografia}: Clasificación de audiencia con soporte de color hexadecimal.
    \item \texttt{Estado}: Ciclo de vida del contenido.
    \item \texttt{Chapter}: Capítulos vinculados a cada Manga con arrays de páginas en JSON.
\end{itemize}

\subsection{Políticas de Integridad}
\begin{itemize}
    \item \textbf{Integridad Referencial:} Restricciones y cascadas configuradas en SQLAlchemy.
    \item \textbf{Transacciones:} Se manejan mediante \texttt{SessionLocal} y operaciones de commit/rollback explícitas.
\end{itemize}
