\section{Manual de Operaciones y Despliegue}

\subsection{Ciclo de Vida del Software (SDLC)}
El desarrollo sigue un flujo ágil e iterativo:
1. \textbf{Desarrollo Local:} API testeada con \texttt{uvicorn} y frontend con \texttt{Vite}.
2. \textbf{Gestión de Paquetes:} Entorno manejado con \texttt{pip} / \texttt{uv} para el backend y \texttt{npm} para el frontend.
3. \textbf{Build Backend:} Empaquetado y despliegue automático en Railway u otro PaaS.
4. \textbf{Migraciones de DB:} Se aplican los cambios de esquema mediante \texttt{alembic upgrade head}.
5. \textbf{Protección CDN:} Despliegue de reglas en Cloudflare Workers usando \texttt{wrangler deploy}.

\subsection{Monitorización y Alertas}
\begin{itemize}
    \item \textbf{Disponibilidad:} Endpoint \texttt{/api/health} revisa el estado de conexión a la base de datos MySQL.
    \item \textbf{Errores API:} Registro directo mediante las excepciones de FastAPI.
    \item \textbf{Logs:} Logs asíncronos de uvicorn accesibles directamente desde el panel del proveedor cloud.
\end{itemize}

\subsection{Procedimientos de Recuperación (Disaster Recovery)}
\subsubsection{Restauración de Servicio}
En caso de fallo crítico en el despliegue del backend:
1. Acceder al Dashboard del PaaS (ej. Railway).
2. Seleccionar el commit estable anterior y ejecutar un "Rollback".
3. El contenedor revertirá su imagen en pocos segundos.

\subsubsection{Backup de Datos}
La base de datos MySQL cuenta con backups gestionados automáticamente por el proveedor de alojamiento.
