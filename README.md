# MangaAPI Project (v2.0)

Modern full-stack platform for managing and reading manga/chapters.

## 🌟 Architecture Overview (Ponytail Architecture)

The project has recently migrated to a highly optimized, lean stack:

- **FastAPI Backend (`MangaApi/`)**: High-performance asynchronous API using Python 3.13, SQLAlchemy 2.0, Alembic for migrations, and Pydantic for validation. It manages data models (mangas, chapters, demografías, tags, autores, estados), OAuth2 JWT authentication, and admin endpoints.
- **Vue 3 + Vite Frontend (`MangaClient/`)**: Minimalist, fast UI built with the Composition API (without Pinia/Vuex for global state). It handles browsing, catalog filtering, chapter reading (two-page 'libreta' mode), and an admin upload area.
- **Cloudflare Edge Security**: A Cloudflare Worker sits in front of the Blackblaze B2 bucket to provide robust anti-hotlink protection (validating `Origin`/`Referer`).
- **Python Utility Framework (`MangaFramework/` & `MangaScripts/`)**: Standalone scripts for scraping, mass image downloading, renaming, WEBP conversion, and JSON payload generation.

## 1. Quick Start

### 1.1 Clone & Enter Project

```powershell
git clone <repo-url> MangaAPI
cd MangaAPI
```

### 1.2 Create & Activate Virtual Environment

```powershell
uv venv
# or python -m venv venv
venv\Scripts\Activate.ps1
```

*(If execution policy blocks activation on Windows: `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`)*

### 1.3 Install Backend Dependencies

```powershell
pip install -r requirements.txt
```

### 1.4 Environment Variables

Create a `.env` file in the root of `MangaApi/`. Typical variables needed for FastAPI and SQLAlchemy:

```env
DATABASE_URL=sqlite:///./manga_local.db  # Use mysql+aiomysql://user:pass@host/db in production
SECRET_KEY=your-secure-random-secret-key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
BACKBLAZE_BASE_URL=https://f000.backblazeb2.com/file/TuBucket
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

### 1.5 Apply Migrations

```powershell
alembic upgrade head
```

### 1.6 Run Backend Server (FastAPI)

```powershell
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

The FastAPI autogenerated interactive docs will be available at:

- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`

### 1.7 Run Frontend (Vue Client)

Open a new terminal:

```powershell
cd MangaClient
npm install
npm run dev
```

Vite dev server defaults to `http://localhost:5173/`.

## 2. Project Structure

```text
MangaAPI/
├── core/                # FastAPI settings, security, and database session
├── routers/             # API Endpoints (auth, manga, chapters, admin)
├── schemas/             # Pydantic models (data validation)
├── models/              # SQLAlchemy ORM models
├── services/            # Business logic (DB operations decoupled from routers)
├── alembic/             # Database migration scripts
├── MangaClient/         # Vue 3 Frontend (Vite)
├── MangaFramework/      # Reusable scraping/processing tools
└── RedirectionBlackblaze/ # Cloudflare Worker scripts
```

## 3. Upload Chapter Flow

The admin interface expects chapters to be pre-processed (downloaded, converted to `.webp`, and uploaded to B2).

Example JSON payload sent to the backend (`POST /api/chapters/`):

```json
{
  "manga_id": 12,
  "number": "005",
  "title": "Comienzo del Arco",
  "pages": [
    "https://img.tusitio.com/file/MangaApi/chapters/ABC123/005/001.webp",
    "https://img.tusitio.com/file/MangaApi/chapters/ABC123/005/002.webp"
  ]
}
```

## 4. Authentication (OAuth2)

- The frontend authenticates via the standard OAuth2 `POST /api/auth/token` endpoint.
- On success, the JWT token is stored securely in the client (Memory/LocalStorage).
- Protected API routes utilize FastAPI's `Depends(get_current_user)` or `Depends(get_admin_user)` to enforce Role-Based Access Control (RBAC).

## 5. Cloudflare Worker (Anti-Hotlink)

To protect your B2 bucket from direct access and embedding in external domains, deploy the provided worker in `RedirectionBlackblaze/`:

```powershell
cd RedirectionBlackblaze
npm install
npx wrangler deploy
```

*Ensure your `wrangler.toml` is configured with your B2 bucket details.*
