\section{Arquitectura del Sistema}

\subsection{Visión General}
MangaReader es una plataforma de distribución de contenido digital (manga/manhwa) construida sobre una arquitectura "Ponytail" ligera y modular con FastAPI. El sistema está diseñado para alta disponibilidad, baja latencia, escalabilidad horizontal y seguridad robusta.

\subsection{Tecnologías Principales}
\begin{itemize}
    \item \textbf{Backend:} Python 3.13 + FastAPI.
    \item \textbf{ORM y Validación:} SQLAlchemy, Alembic y Pydantic.
    \item \textbf{Base de Datos:} MySQL (Producción), SQLite (Desarrollo).
    \item \textbf{Infraestructura:} Cloudflare Workers (Protección B2/Hotlink) y Railway (PaaS).
    \item \textbf{Frontend:} Vue.js 3 (Composition API) optimizado sin gestores de estado complejos (sin Pinia).
\end{itemize}

\subsection{Diagrama de Componentes}
El sistema se divide en los siguientes módulos lógicos:
\begin{enumerate}
    \item \textbf{Routers / Endpoints:} Puntos de entrada HTTP manejados por FastAPI (ej. \texttt{catalog}, \texttt{reader}).
    \item \textbf{Servicios (Services):} Lógica de negocio encapsulada y separada de las rutas.
    \item \textbf{Capa de Datos:} SQLAlchemy gestiona los modelos (\texttt{Manga}, \texttt{Chapter}, \texttt{User}).
    \item \textbf{Seguridad:} Dependencias de FastAPI inyectadas para verificar tokens OAuth2 JWT.
    \item \textbf{Cache/CDN:} Resoluciones de URL de imágenes apuntando directamente al CDN B2 respaldado por Cloudflare Workers.
\end{enumerate}

\subsection{Flujo de Datos}
1. El cliente solicita un token JWT vía \texttt{/api/auth/token}.
2. Las peticiones subsiguientes incluyen el header \texttt{Authorization: Bearer <token>}.
3. Los Middlewares (ej. CORS, Throttling) y Dependencias interceptan la petición.
4. El servicio correspondiente (ej. \texttt{MangaService}) recupera datos de la base de datos usando SQLAlchemy.
5. Pydantic serializa la respuesta, optimizando y validando los datos salientes de forma síncrona/asíncrona.
