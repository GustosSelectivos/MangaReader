\section{Protocolos de Seguridad}

\subsection{Control de Acceso (RBAC)}
El sistema implementa un control de acceso basado en roles mediante Dependencias de FastAPI:

\begin{enumerate}
    \item \textbf{Anónimo:} Lectura de catálogo general y lectura de capítulos.
    \item \textbf{Registrado (\texttt{get\_current\_user}):} Acceso a marcadores y lectura restringida.
    \item \textbf{Verificado (+18):} Acceso a NSFW filtrado a nivel de serialización/base de datos.
    \item \textbf{Admin (\texttt{get\_admin\_user}):} Permisos de escritura y acceso a endpoints de administración.
\end{enumerate}

\subsection{Autenticación}
Se utiliza el estándar **OAuth2 con JWT (JSON Web Token)**.
\begin{itemize}
    \item \textbf{Access Token:} Firmado criptográficamente (HS256). Vida útil configurable (ej. 30 minutos).
    \item \textbf{Almacenamiento:} El cliente (Vue) guarda el token en memoria o \texttt{localStorage} y lo anexa en el header \texttt{Authorization: Bearer}.
\end{itemize}

\subsection{Protección de Infraestructura}
\subsubsection{Rate Limiting}
Implementado mediante dependencias o middlewares de Throttling personalizados para FastAPI:
\begin{itemize}
    \item Endpoints públicos: Límites más estrictos para frenar scraping.
    \item Rutas autenticadas: Límites extendidos según rol.
\end{itemize}

\subsubsection{Protección de CDN (Hotlink Protection)}
Los recursos estáticos pesados (imágenes almacenadas en Blackblaze B2) están protegidos mediante un Worker de Cloudflare en Edge. Éste verifica el header \texttt{Referer} y el \texttt{Origin}, bloqueando todo acceso directo no autorizado e incrustaciones desde dominios externos maliciosos.

\subsection{CORS}
Políticas estrictas gestionadas por el \texttt{CORSMiddleware} de FastAPI, permitiendo solicitudes provenientes únicamente de orígenes pre-aprobados (ej. el frontend en producción y \texttt{localhost} en desarrollo).
